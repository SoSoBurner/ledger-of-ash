# V32_4 Release Notes

This release maxes out previously thin support layers by replacing placeholder locality helper packets, institution rosters, and NPC roster cards with materially richer repository-backed content.

## What changed
- Expanded thin arrival, inside, seasonal, authority, behavior, and pressure packets where placeholder text had been standing in for playable support.
- Rebuilt small institution rosters from repository named NPC records, linked systems, and locality identity context.
- Rebuilt thin NPC roster cards so they now expose usable named figures, role tags, and local context instead of count-only placeholders.
- Preserved canon boundaries by deriving from existing repository objects and named NPC files rather than inventing unsupported major canon.

## Build intent
V32_4 is a thin-area saturation pass aimed at improving runtime usability, locality readability, and GM-facing quick reference depth while remaining standalone.
