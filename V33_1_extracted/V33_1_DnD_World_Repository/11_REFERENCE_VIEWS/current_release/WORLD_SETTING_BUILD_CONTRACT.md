V33_1 WORLD-SETTING DEPTH BUILD CONTRACT

Authoritative build objective:
This document records the world-setting depth build logic that informed V33_1. It must not be treated as a broad lore expansion pass, campaign-state expansion pass, or generalized content-growth pass. It must be treated as the release where the asserted world setting is deepened into a denser operational environment through rumor architecture, institution interiors, route ecology, locality-cluster integration, named NPC saturation, law-and-behavior texture, culture-in-practice surfaces, and stronger reference synthesis while preserving canon integrity and repository standalone usability.

Primary release identity:
current release = World-Setting Depth, Canon-Safe Densification, Operational Locality Expansion Release.

Core doctrine:
The purpose of current release is not to widen the world. The purpose is to deepen the already asserted world so that places, institutions, routes, and locality clusters behave as stronger world-setting surfaces without depending on live campaign state, external explanation, or speculative canon invention.

current release must preserve and operationalize the following top-level conclusion:
The world setting does not most need new broad lore, new major polities, new cosmology, new top-level history, or ornamental expansion. It most needs deeper authored support inside already asserted world structures so that the setting becomes denser, more internally connected, more socially legible, more route-legible, and more immediately usable at the table.

Non-goal boundary:
current release is not a live campaign continuity release.
current release is not a current-session packet release.
current release is not a party-specific progression release.
current release may reference existing runtime materials where necessary for crosslink safety, but the build focus must remain the world setting layer itself.

Standalone package requirement:
The finished artifact must remain fully standalone after unzip and must not require outside dependencies, browser access, or off-package explanation in order to function as a complete world-setting repository package.

False condition:
FALSE = the artifact may mention that it was dependent on outside materials or off-package context.

Failure condition:
Response = Failed, IF = the output does not create files that function complete stand alone within a .zip, OR if the build widens unsupported canon instead of deepening asserted canon.

WORLD-SETTING BUILD MANDATE

current release must prioritize the following workstreams in descending order of importance:

1. Rumor architecture expansion
2. Institution interior deepening
3. Route ecology expansion
4. Locality-cluster packetization
5. Named NPC saturation in world-facing places
6. Local law and behavior surfaces
7. Culture-in-practice packets
8. World reference synthesis views

The build must explicitly treat these as canon-safe deepenings of already asserted world structures rather than as new truth-layer invention.

REQUIRED CONCLUSIONS TO PRESERVE

The following conclusions are mandatory and must govern the build:

1. The world-setting layer is already broadly established.
The next highest-value work is not more generalized worldspread. It is deeper operational texture of asserted places, institutions, routes, cultures, and social behavior.

2. Rumor coverage remains one of the thinnest world-setting layers.
current release must treat rumor packs as a primary build target because they make existing canon socially circulate without forcing hard retcons.

3. Institution interiors remain under-authored relative to locality surface coverage.
current release must deepen what happens when characters move from approach and arrival into civic, legal, sacred, logistical, and industrial interiors.

4. Routes must be treated as lived corridors rather than abstract connectors.
current release must expand travel ecology, corridor behavior, waypoint logic, checkpoint logic, freight movement, and authority visibility.

5. Locality clusters must be treated as functioning territorial systems.
current release must deepen relationships between core localities and their feeders, spillover zones, linked institutions, and route traffic.

6. Named NPC saturation remains uneven.
current release must replace thin or overly generic world-facing surfaces with more asserted or conservative-inference named anchors who make places feel socially continuous.

7. Local law, behavior, and culture-in-practice need stronger locality-level manifestation.
current release must show how abstract polity identity becomes visible in greetings, dress, transaction norms, shrine behavior, checkpoint behavior, authority expectations, suspicion triggers, and outsider treatment.

8. Reference synthesis views remain high value.
current release must produce additional retrieval-safe views that help users compare regions, institutions, route types, and behavioral differences without flattening canon distinctions.

- enter

- operationally run
- compare to nearby places

Each world-setting artifact must include, where applicable:
- stable object identity
- truth-layer classification
- hierarchy path

MANDATORY TRUTH-LAYER DISCIPLINE

Every artifact must classify its content as one of the following:
- direct canon
- corrected canon
- reconciled canon
- conservative inference
- unresolved contradiction
- deprecated or superseded legacy material

No artifact may silently convert conservative inference into asserted hard canon.
No artifact may silently harmonize preserved contradictions.
No artifact may introduce unsupported major history, factional truth, metaphysical law, or geopolitical expansion.

MANDATORY BUILD LOGIC

current release must follow this order of reasoning:

1. Harvest asserted world objects already present in the repository.
2. Classify support thinness without widening canon.
3. Deepen the thinnest high-value asserted surfaces first.
4. Crosslink new depth artifacts back into existing localities, routes, factions, and named people.
5. Validate that the package remains standalone, repository-governed, and canon-safe.

current release must prefer deepening over expansion in every ambiguous case.

If forced to choose between:
- adding a new place, or
- making an already asserted place socially, institutionally, and route-wise operable,

current release must choose the second.

QUALITY STANDARD

A world-setting artifact is successful only if it strengthens one or more of the following without drifting canon:
- approach clarity
- interior usability
- route legibility
- social readability
- institutional specificity
- cross-region differentiation
- retrieval reliability
- table readiness

Artifacts that only add flavor language without improving operational usability do not satisfy the build objective.
