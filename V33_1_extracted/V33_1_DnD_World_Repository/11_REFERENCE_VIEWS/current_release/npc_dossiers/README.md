# NPC Dossiers

This surface provides one dossier per named NPC in the package, expanded from the baseline NPC architecture into a world-usable life packet without silently promoting unsupported facts.

## Use rules
- Treat each dossier's public layer as canon-grounded.
- Treat hidden sections as conservative connective tissue derived from existing role, locality, pressure, and relationship support.
- If a hard biographical field is unsupported, it remains explicitly unstated rather than invented.
