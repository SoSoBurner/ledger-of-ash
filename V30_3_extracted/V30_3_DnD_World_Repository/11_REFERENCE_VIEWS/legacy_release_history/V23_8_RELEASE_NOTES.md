# lineage_release Release Notes

lineage_release is a canon-grade trust and retrieval hardening full release built from the uploaded prior repository package and the source corpus already represented inside the repository.

## Summary

This release replaces shallow validation theater with audited release reports, corrects live inventory drift, purifies the active namespace, deepens provenance measurement, and connects reusable interface roles and named NPC interfaces to explicit world contexts.

## Why lineage_release needed patching

- several active files still carried stale lineage_release identifiers
- some active reports still miscounted live files
- provenance and unsupported-inference validations reported pass states without auditing real active objects
- interface roles and some NPC interfaces remained structurally under-contextualized

## Files changed

- active package metadata and namespace references
- 00_PACKAGE_MANIFEST/object_count_report.json
- 00_PACKAGE_MANIFEST/package_manifest.json
- 00_PACKAGE_MANIFEST/filesystem_inventory_report.json
- 05_PROVENANCE/object_to_source_map.jsonl
- 05_PROVENANCE/derivation_justifications.jsonl
- interface role and NPC interface objects under 02_CANON_BASELINE
- regenerated validation reports under 06_VALIDATION
- README.md and current release manifest metadata

## Validation changes

- counts now derive from a single live filesystem inventory
- version consistency checks fail stale active namespace bleed instead of ignoring it
- provenance coverage and unsupported-inference reports audit real active objects
- release consistency and gold-master checks compare active reports against discovered filesystem truth
- orphan and interface parity checks now evaluate contextual attachment rather than mere folder presence

## Remaining limitations

This release substantially improves trust and auditability, but unresolved canon contradictions intentionally remain preserved, and some engine-canon synthesis layers still depend on future source growth for deeper direct-canon replacement.
