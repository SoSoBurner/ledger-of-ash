# V33_1 Psycho-Social Chronology Build Method

This layer is additive. It does not overwrite baseline canon.

## Purpose
V33_1 upgrades the repository by treating each NPC as a living entity embedded in routine, hierarchy, memory, scarcity, and obligation. Historical explanation is generated by passing existing NPC records through a psycho-social interpretation layer and then surfacing only canon-safe residue.

## Operating discipline
1. Preserve the existing canon surface.
2. Generate support material beside it.
3. Keep inferred material visibly inferential.
4. Promote nothing silently.
5. Treat historical outcomes as products of people, not just dates.

## Living-entity read standard
Each NPC is interpreted through:
- daily routine
- public posture versus private pressure
- status position
- economic exposure
- authority reach
- moral posture and alignment tendency
- risk tolerance
- fear and desire
- sacrifice thresholds
- never-surrender values

## Build products
- `npc_living_entity_profiles.jsonl`
- `npc_psycho_social_chronology_registry.jsonl`
- `historical_change_registry.json`
- `antagonist_dossier_crosswalk.json`

## Canon safety
These files explain how present canon likely became itself. They are not replacements for baseline dossier truth.
