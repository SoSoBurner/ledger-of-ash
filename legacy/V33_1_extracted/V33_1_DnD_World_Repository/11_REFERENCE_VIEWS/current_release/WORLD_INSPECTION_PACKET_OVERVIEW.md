# World Inspection Packet Overview

Inspection packets are a deeper truth-status and contradiction-review layer intended to support prep and adjudication, not replace the primary play-first entry flow.

## What this layer covers
- Localities: 54
- Polities: 16
- Macroregions: 15

## Where to browse
- `world_inspection_packets/localities/`
- `world_inspection_packets/polities/`
- `world_inspection_packets/macroregions/`

## Authoritative contract
- `../../01_CANON_GOVERNANCE/world_inspection_packet_contract.json`

## How to use it well
1. Start from `../../RUN_A_SESSION_NOW.md` or `../../START_HERE_GM.md`.
2. Use starter packets and table-ready locality cards for first-scene prep.
3. Open an inspection packet only when you need canon-confidence support, unresolved-field visibility, or contradiction review.
4. Preserve uncertainty where the packet marks uncertainty instead of hardening it into false canon.

## Why it exists
This layer makes explicit what is fully asserted, what is inferred, and what remains unresolved so deeper prep and runtime adjudication stay honest to the shipped package.
